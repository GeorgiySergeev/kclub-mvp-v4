## Findings

The intended architecture is confirmed as one Bun/Turbo monorepo with:

- `apps/product-core` for public/member UX, product APIs, admin APIs, Stripe webhooks, cron, and product business logic.
- `apps/admin-app` for staff UX, staff auth/session shell, and proxy access to product-core admin APIs.
- `packages/*` for contracts, validation, domain policies, database artifacts, shared UI primitives, config, and test utilities.

Split-repo language found in `docs/BLUEPRINT.md` and related docs is framed as migration guidance or guardrails, not as current implementation direction. The current source of truth explicitly rejects stale split-repo assumptions such as separate `kclub-product-core` and `kclub-admin-app` repositories.

Drift found:

- `docs/development/phase-0/step-p0.1-docs-and-scope-audit.md` asks agents to read `docs/review.md`, but that file does not exist.
- `docs/BLUEPRINT.md` includes `docs/review.md` in its example repository layout, but the actual docs tree has no such file.
- Root `README.md` is directionally aligned with the monorepo plan, but Phase 1 should refresh it after scaffold decisions are materialized so the README reflects real scripts, pinned Bun version, workspace commands, and any generated app/package locations instead of only planned architecture.

## Open Product Defaults

Use these defaults from `docs/SPEC.md` unless Product changes them before implementation:

- MEMBER to VIP card re-issue: keep existing `MEM-*`; admin can manually re-issue `VIP-*`.
- Multiple introductions to same target: allow one pending introduction at a time; apply a 30-day cooldown after terminal state.
- Canceled VIP access: capabilities remain active until `current_period_end`.
- Business draft mode: not in MVP.

## Implementation Notes

Later implementation prompts that conflict with this audit should pause and reconcile docs before making product, security, billing, migration, or irreversible data changes.

Do not start Phase 1 from split-repo assumptions. Scaffold inside this repository using `apps/product-core`, `apps/admin-app`, and shared `packages/*`.
